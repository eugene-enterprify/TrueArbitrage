# TrueArbitrage

Монітор арбітражних спредів між USDT-перпетуалами **Binance Futures** і **OKX Swap**.
Відстежує всі спільні пари через WebSocket, рахує чистий спред (комісії + фандинг),
пише історію в SQLite і надсилає сповіщення в Telegram, коли спред перевищує поріг.

Стратегія: long на біржі з нижчою котировкою + short на біржі з вищою, вихід при сходженні.

Детальний план і архітектура: [PLAN.md](PLAN.md)

## Запуск

```powershell
# 1. Створити venv і встановити залежності (один раз)
python -m venv venv
.\venv\Scripts\python.exe -m pip install -r requirements.txt

# 2. Покласти токен бота у .env (файл не комітиться)
#    TELEGRAM_BOT_TOKEN=123456:ABC...

# 3. Запустити
.\venv\Scripts\python.exe main.py
```

Після запуску надішліть боту `/start` — він запам'ятає ваш чат і почне слати сповіщення.

## Команди бота

| Команда | Опис |
|---|---|
| `/start` | реєстрація чату |
| `/status` | стан з'єднань, кількість пар, відкриті спреди |
| `/top` | топ-10 поточних спредів (з фільтром обсягу) |
| `/threshold 2.5` | поріг спреду у % |
| `/minvolume 20` | мінімальний обсяг 24h у млн $ |
| `/mute BTC` / `/unmute BTC` | вимкнути/увімкнути сповіщення по монеті |
| `/export 24` | CSV-файл з історією за N годин |

## Налаштування

Дефолти — у [config.yaml](config.yaml). Значення, змінені командами бота,
зберігаються в базі та мають пріоритет після перезапуску.

## Структура

- `src/exchanges/` — адаптери бірж (нова біржа = новий файл за інтерфейсом `base.ExchangeAdapter`)
- `src/spread_engine.py` — розрахунок спредів (bid/ask, нормалізація множників 1000x, фільтр застарілих котирувань)
- `src/alert_engine.py` — поріг, cooldown, події відкриття/закриття спреду
- `src/telegram_bot.py` — сповіщення і команди (aiogram)
- `src/storage.py` — SQLite-історія із семплінгом, CSV-експорт
