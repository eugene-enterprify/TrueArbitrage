# Як запустити монітор після вимкнення комп'ютера

## Найпростіший спосіб — подвійний клік

Відкрийте папку `C:\Work\Projects\cross-exchange-arbitrage` і двічі клікніть на файл:

```
start.bat
```

Відкриється чорне вікно з логами — це і є монітор. Поки вікно відкрите, монітор працює.

## Через термінал (альтернатива)

Відкрийте PowerShell і виконайте:

```powershell
cd C:\Work\Projects\cross-exchange-arbitrage
.\venv\Scripts\python.exe main.py
```

## Як зрозуміти, що все запустилося

Через 5–10 секунд у вікні з'являться рядки:

```
Binance: 659 USDT-перпетуалів
OKX: 405 USDT-перпетуалів
Спільних монет: 360
Монітор запущено. ...
Binance: з'єднано
OKX: з'єднано
```

Після цього надішліть боту `/status` — обидві біржі мають бути 🟢.

`/start` заново надсилати НЕ потрібно — ваш чат збережений у базі назавжди.

## Як зупинити

Натисніть `Ctrl+C` у вікні монітора або просто закрийте вікно.

## Важливі правила

1. **Тільки одна копія одночасно.** Якщо запустити дві, Telegram-бот почне конфліктувати
   (у лозі буде помилка `TelegramConflictError`). Перед повторним запуском переконайтесь,
   що старе вікно закрите.
2. **Не видаляйте файл `data/spreads.db`** — там уся накопичена історія спредів
   і реєстрація вашого чату.
3. **Не видаляйте файл `.env`** — там токен бота.
4. Налаштування, змінені через бота (`/threshold`, `/minvolume`, `/mute`),
   зберігаються і після перезапуску.

## Якщо щось пішло не так

- Вікно одразу закрилося або сипле помилками — запустіть через PowerShell
  (другий спосіб вище), щоб побачити текст помилки, і надішліть його в чат агенту.
- Бот не відповідає — перевірте, що монітор запущений і в лозі немає помилок.
- `/status` показує 🔴 — зачекайте хвилину (авто-перепідключення) або перезапустіть монітор.
